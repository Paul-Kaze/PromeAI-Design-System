# PromeAI Landing Page Design

Use this skill when the user wants a PromeAI-flavored landing page, page redesign, campaign mockup, visual prototype, or production UI treatment.

## Workflow

1. Identify the page goal, audience, conversion action, and product/workflow being promoted.
2. Read `references/style-guide.md` for the reusable visual language.
3. Use `colors_and_type.css` for tokens when writing HTML/CSS or adapting a frontend codebase.
4. Use `preview/` for visual specimens and `ui_kits/landing/` as optional component references.
5. Replace all placeholder copy, imagery, proof points, and flows with the current page context.

## Design Direction

Create a dark, premium, AI-product landing page with electric blue and vivid purple accents, Poppins typography, clean slate cards, restrained gradients, and practical conversion structure. The skill should guide the look and interaction style only; it must not force a specific product, feature, headline, workflow, or offer.

## Output Expectations

- For prototypes, produce a viewable static HTML or React page using local assets when useful.
- For production work, follow the existing app framework and component patterns.
- Keep copy specific to the user's actual offer. Do not reuse bundled demo text as final content.
- Keep CTAs, screenshots, examples, FAQ, and proof points aligned to the current page.
